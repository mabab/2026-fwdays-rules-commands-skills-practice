# Markdown Rules

## Document Structure

- Use headers for hierarchical structure
- Add table of contents for long documents
- Use horizontal lines (---) to separate sections

## Text Formatting

- Use bold text for important terms
- Use italic for emphasis
- Use code format for technical terms and commands
- Add links with descriptive text

## Lists

- Use bullet lists for enumerations
- Use numbered lists for step sequences
- Nest lists for sub-items (maximum 2-3 levels)

## Diagrams and Visualizations

- Use Mermaid for diagrams when possible
- Add descriptions to diagrams for accessibility
- Test diagrams before finalization
- Use PlantUML only when Mermaid doesn't fit

## Tables

- Use tables for structured data
- Align columns for readability
- Add headers to tables

## Markdown Best Practices

- Keep line length reasonable (80-100 characters)
- Use consistent heading levels
- Add alt text for images
- Use reference-style links for long URLs
