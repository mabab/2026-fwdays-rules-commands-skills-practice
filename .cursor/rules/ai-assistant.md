# AI Assistant Rules

## Query Formulation

- Be specific and clear in queries
- Provide context for complex tasks
- Specify expected results
- Use examples for illustration

## Iterative Improvement

- Review results before accepting
- Ask clarifying questions
- Suggest alternative solutions
- Learn from each interaction

## Critical Evaluation

- Analyze proposed code
- Check for errors and vulnerabilities
- Evaluate quality and readability
- Compare with best practices

## Working with AI Assistant

- Break down complex tasks into smaller steps
- Provide relevant context and examples
- Review and test AI-generated code
- Understand the reasoning behind suggestions
- Don't blindly accept all suggestions

## Best Practices

- Use AI as a tool to enhance productivity, not replace critical thinking
- Always review and understand generated code
- Test AI suggestions thoroughly
- Maintain code quality standards regardless of source
