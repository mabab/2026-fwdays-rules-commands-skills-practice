# Slidev Rules

## Slide Structure

- Each slide should have a clear purpose
- Use frontmatter for slide configuration
- Follow the sequence: introduction → main content → conclusions
- Add transitions between slides for smoothness

## Markdown for Slides

- Use headers for content structuring
- Add empty lines between sections for readability
- Use lists for enumerating points
- Use emphasis (bold, italic) for highlighting

## Code in Slides

- Use syntax highlighting for all code blocks
- Specify programming language in code blocks
- Use line highlighting to emphasize important lines
- Add explanations for complex code fragments

## MDC Syntax

- Use MDC components for interactivity
- Follow correct syntax for v-click, v-motion, etc.
- Test interactive elements after creation

## Slidev Best Practices

- Keep slides focused and concise
- Use visual elements to support content
- Ensure readability on different screen sizes
- Test presentation flow before finalizing
