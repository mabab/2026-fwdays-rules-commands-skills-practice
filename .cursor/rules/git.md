# Git and Versioning Rules

## Commits

- Use meaningful commit messages
- Group related changes in one commit
- Write commit messages in English
- Follow format: "type: short description"

### Commit Types

- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation changes
- `style`: Code style changes (formatting, etc.)
- `refactor`: Code refactoring
- `test`: Adding or updating tests
- `chore`: Maintenance tasks

## Versioning

- Update version for significant changes
- Document breaking changes
- Maintain changelog for important updates

## Additional Recommendations

### Productivity

- Use quick commands for common tasks
- Create templates for repetitive structures
- Automate routine tasks
- Optimize workflow

### Collaboration

- Follow team conventions
- Communicate changes with colleagues
- Share useful findings
- Help others learn

### Learning

- Experiment with new approaches
- Study best practices
- Adapt rules to specific needs
- Continuously improve skills

## Git Best Practices

- Commit often with clear messages
- Use branches for features and fixes
- Review changes before committing
- Keep commits focused and atomic
- Write descriptive commit messages
