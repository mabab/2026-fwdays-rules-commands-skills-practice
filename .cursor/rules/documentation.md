# Documentation Rules

## PRD Document

- Follow the structure defined in docs/PRD.md
- Update PRD when requirements change
- Use clear and specific formulations
- Add examples for complex concepts

## README

- Always update README when adding new features
- Add installation and startup instructions
- Include usage examples
- Add links to important resources

## Code Comments

- Comment "why", not "what"
- Remove commented code before committing
- Use English for comments
- Add TODO comments with context

## Code Documentation

- Add JSDoc for public functions
- Document complex algorithms
- Explain non-obvious solutions
- Add usage examples

## Documentation Best Practices

- Keep documentation up to date with code
- Use clear and concise language
- Include code examples where helpful
- Document edge cases and limitations
