# Project Context and General Rules

## Project Context

This project is a workshop about rules and skills for working with AI assistants, created using Slidev. The project includes presentation slides, documentation, and code examples.

## General Rules

### Language and Communication
- Always respond in English
- Use professional but understandable terminology
- Explain technical terms when necessary
- Maintain a friendly and supportive tone

### Code Style
- Use 2 spaces for indentation in all files
- Always add semicolons in JavaScript/TypeScript
- Use single quotes for strings in JavaScript/TypeScript
- Maintain consistent code formatting
- Use meaningful names for variables and functions

### File Structure
- Maintain logical project structure
- Group related files together
- Use clear file and folder names
- Follow project conventions

## Important Reminders

1. **Always check code before committing** - run linter and tests
2. **Follow project structure** - keep files in the right places
3. **Update documentation** - synchronize code changes with documentation
4. **Test interactive elements** - check slides before presentation
5. **Be specific** - clear instructions yield better results

## Context for AI Assistant

When working with this project:
- Pay attention to docs/PRD.md for understanding requirements
- Use slides.md as the main file for work
- Follow the structure and style of existing slides
- Add interactive elements to engage the audience
- Focus on practical examples and cases
- Ensure balance between theory and practice

---

*These rules are designed to ensure quality, consistency, and efficiency in working on the workshop project.*
